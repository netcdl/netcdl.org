The Network Certification Description Language (NETCDL)

NETCDL Project Homepage

#Use NETCDL to make assertions about how you want your network port to behave.

#call out the particular network interface we want this certification description to apply to.
interface CoreSwitch-3600/GigabitEthernet0/0/1

#assertions about the physical layer
link should be 1Gb
link should be full duplex
link medium should meet standard ANSI/TIA-568-C.1

#also for Wireless
channel should be 6
ssid ‘my-wifi-ap’ should be heard
transmit mode should be 802.11ac

#assertions for the link (or ethernet) layer
#Declare a vlan
vlan should be 500
#Declare traffic from VLAN's that should not be seen here.
frames from vlan 600 should not be seen
#Or types of frames that shouldn't be seen
cdp frames should not be seen
netBIOS frames should not be seen
#Also assert the positive case, that we SHOULD see a type of frame.
expect frames from vlan 100

#network layer
#Blacklist IP networks
packets from network 10.0.0.0/24 should not be seen
#Or Hosts
packets from host 10.1.1.1 should not be seen
#Assert that other networks or hosts should be reachable
google.com should be reachable

#network services
#Assert that the DHCP config behaves as expected
dhcp server should be 192.168.1.1
dhcp network should be 192.168.1.0/24
dhcp default gateway should be 192.168.1.1
dhcp dns servers should be 8.8.8.8, 8.8.4.4
#Or other network services
ntp server should be 192.168.1.2

#application layer
#Ensure connectivity on certain ports
host 10.20.20.1 should be reachable on tcp port 80, 443
#Or even application layer tests
web server 10.20.20.1:80/welcome.html should serve html with pattern /welcome (.*)!/
#Check security by asserting that certain hosts and ports should NOT be accessible
host 10.20.20.1 should not be reachable on tcp port 22
host 10.20.20.2 should be reachable by ping
host 10.20.20.3 should not be reachable by ping
#make assertions about a particular flow
packets from this interface to 10.20.20.100 should be encrypted
